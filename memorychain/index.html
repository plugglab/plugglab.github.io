Capabilities

What MemoryChain does

MemoryChain doesn't add gameplay features on its own — it gives every other PluggLAB plugin a shared foundation to build on.

Persistent Player Memory

Player data, preferences, and cross-plugin flags persist reliably across restarts and reloads.

Cross-Plugin Data Sharing

Casino Core, SafeTrade, and PulseEvents all read and write through the same MemoryChain layer instead of duplicating storage logic.

Optimized Storage

Batched writes and in-memory caching keep disk I/O low even on busy servers with hundreds of concurrent players.

One Update Channel

Every PluggLAB plugin checks MemoryChain for shared config versions, so update rollouts stay consistent across the whole stack.

Safe Migrations

Schema changes migrate existing player data automatically — no manual re-entry needed after an update.

Developer API

A lightweight API lets future PluggLAB plugins hook into shared player memory without redefining their own storage layer.
